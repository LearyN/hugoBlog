没有 K8s，没有 Docker Compose，没有编排框架。一个 `workspace/` 目录 + 四个 Python 脚本 + Agent 框架的 tools routing，就能让 OpenClaw 替你盯盘、出评、推卡片。

## 架构

```text
用户（飞书消息）← → OpenClaw Gateway
                         │
            （toolsLayer 路由）
                │
      ┌─────────┼─────────┬─────────┐
      ▼         ▼         ▼         ▼
 Tushare Pro  腾讯行情  财联社情绪  可转债
      │         │         │         │
      └─────────┴────┬────┴─────────┘
                     ▼
              Python 脚本库（4 脚本）
                     │
                     ▼
           飞书消息卡片（Schema 2.0）
```

## 文件布局

```text
workspace/
├── scripts/
│   ├── morning_report.py      # 早评（隔夜外盘 + 昨日复盘 + 板块判断 + 潜力票池 + 持仓要点）
│   ├── midday_report.py       # 午评（上午实盘验证 + 午后策略微调）
│   ├── evening_report.py      # 收评（全天诊断 + 板块轮动 + 明日策略）
│   ├── auction_watcher.py     # 竞价盯盘（9:15/9:20/9:25 三轮，持仓/热票异动监控）
│   ├── sentiment_news.py      # 财联社快讯拉取 + 板块情绪聚合
│   └── convertible_bond.py      # 可转债双低/折价/强赎信号
├── memory/
│   ├── holdings.md            # 实时持仓（用户飞书口述买卖，OpenClaw 自动写入）
│   └── lessons.md             # 踩坑日志，供 Agent 事后调参
├── skills/
│   └── a-stock-advisor/       # A 股投顾技能包（唯一启用的 skill）
│       └── SKILL.md           # 报告框架 + 思维模型 + 过滤规则集 + 输出协议
├── SOUL.md                    # 身份映射
├── IDENTITY.md                # 人设 + 输出风格约定
├── TOOLS.md                   # 工具注册簿（每条 API 的调用方式、认证方式、典型 query）
└── HEARTBEAT.md               # 心跳循环，控制 OpenClaw 主动找用户还是等用户开口
```

## 数据平台

| 平台 | 用途 | 成本 |
|---|---|---|
| **Tushare Pro** | 大盘行情、个股日线、涨跌停统计 | 2000 积分 / 约 200 元 |
| **腾讯行情 API** `qt.gtimg.cn` | 实时盘口、竞价阶段快照 | 零成本，无认证，纯 `urllib.request` |
| **财联社快讯** | 盘中/盘后情绪因子（`sentiment_news.py`） | 按接入方式 |
| **可转债数据** | 双低、折价、强赎预警（`convertible_bond.py`） | 按数据源 |

Tushare 拿盘后数据和历史趋势，腾讯 API 拿实时快照，财联社补情绪面，可转债脚本补衍生品信号。没有中间缓存层——单用户场景不需要。

## 四个脚本全景

| 脚本 | 时机 | 数据源 | 核心目标 |
|---|---|---|---|
| **早评** `morning_report.py` | 开盘前（约 8:30–9:00） | 腾讯行情 + Tushare + 财联社情绪 + 可转债 | 隔夜全局 → 昨日复盘 → 今日判断 |
| **午评** `midday_report.py` | 中午（11:30–12:00） | 腾讯行情 + 财联社情绪 + 可转债 | 上午实盘验证 → 午后策略调整 |
| **收评** `evening_report.py` | 收盘后（15:00–15:30） | 腾讯行情 + Tushare + 财联社情绪 + 可转债 | 全天诊断 → 板块轮动 → 明日策略 |
| **竞价盯盘** `auction_watcher.py` | 9:15 / 9:20 / 9:25 三轮 | 腾讯行情（极速）+ 可转债（可选） | 持仓竞价异动监控 → 早盘预备 |

全部触发走 Agent 自带的 cron + heartbeat，不需要外部 scheduler。

## 一、早评 — 隔夜到开盘的全景扫描

**板块构成：**

- **隔夜外盘** — 美股四指数（道指 / 标普 / 纳指 / 费城半导体），看外围情绪
- **昨日 A 股复盘** — 四大指数涨跌、涨跌停家数、成交额（Tushare 取数据）
- **今日实时大盘** — 开盘后四大指数实时报价
- **权重龙头一览** — 14 只核心权重股（茅台 / 宁王 / 平安 / 比亚迪 / 招商等）实时涨跌
- **情绪因子** — 跑 `sentiment_news.py` 拉财联社快讯，按板块聚合打分，标出正负情绪最强的板块
- **可转债信号** — 跑 `convertible_bond.py`，看双低 / 折价 / 强赎预警信号
- **板块判断框架** — 基于情绪得分定今日看多 / 看空板块（输出结构已预留）
- **潜力票池 / 持仓操作** — 输出结构已预留，OpenClaw 按 Skill 规则填充具体标的

**作用：** 起床后第一件事——知道晚上发生了什么、昨天发生了什么、今天市场可能怎么走。

## 二、午评 — 上午走完的验证

**板块构成：**

- **实时大盘** — 四大指数（验证早评判断是否正确）
- **权重股** — 同样 14 只权重，看哪些异动
- **上午风格判断** — 自动判断：创业板强 / 主板强 / 窄幅震荡 / 普涨普跌
- **信号分类** — 涨超 1.5% 标 `positive`，跌超 1.5% 标 `negative`，方便快速扫
- **盘中情绪因子** — 上午财联社快讯的情绪聚合
- **可转债信号** — 盘中可转债异动

**作用：** 中午看一眼——「早评说的方向对了没有？中午该调不调策略？」

## 三、收评 — 收盘结算 + 明日预判

**板块构成：**

- **实时大盘** — 四大指数收盘报价
- **权重涨跌一览** — 14 只权重的全天表现，强 / 弱 / 平自动归类
- **涨跌停统计** — Tushare 拉当天涨停 / 跌停家数（情绪温度计）
- **板块轮动判断** — 多维度规则引擎：
  - 科创 + 创业板都强势 → 科技成长领涨
  - 上证弱 + 创业板强 → 极端分化（权重砸盘，题材活跃）
  - 上证涨 + 创业板跌 → 权重护盘，小票承压
  - 其余 → 普涨 / 普跌 / 窄幅震荡
- **预估成交额** — 从量粗估全市场成交
- **情绪因子** — 全天情绪聚合
- **可转债信号**
- **明日策略框架** — 趋势判断 + 明日方向（输出结构已预留）

**作用：** 收盘复盘——今天赚了亏了为什么？情绪好不好？明天怎么打？

## 四、竞价盯盘 — 竞价阶段异动监控

只做一件事：竞价阶段把需要决策的票推到眼前。

### 三轮机制

| 轮次 | 时间 | 特点 |
|---|---|---|
| Stage 1 | 9:15 | 集合竞价第一口价，主力试盘，挂单浮动大 |
| Stage 2 | 9:20 | 最后 5 分钟可撤单的临界点，能看到真实意图 |
| Stage 3 | 9:25 | 开盘价已定，看最终定价和前两轮的偏差 |

### 板块构成

- **四大指数竞价值** — 大盘竞价怎么开
- **持仓竞价分析** — 自动从 `memory/holdings.md` 读持仓，每只给出：
  - 涨跌幅 + 五级提醒：大幅高开 🟢 / 高开 / 平开微涨 / 低开 ⚠️ / 大幅低开 🔴
  - 开盘价、成交量
- **早评热票竞价观察** — 通过 `--pre-hot-codes` 传入当天关注的标的，看竞价表现
- **操作策略建议** — 根据竞价数据自动生成：
  - 低开 → 「观察 5 分钟，不回收先减仓」
  - 高开 → 「看量比，>1.5 持有，无量分批止盈」
  - 第 2、3 轮加入买入参考建议

**作用：** 竞价三分钟有人盯着——全天第一个也是最重要的交易信号窗口。持仓高开低开、今天要不要操作，这轮基本就有数了。

## 四个脚本的配合逻辑

```text
早评 (8:30)
  └─→ 定今日框架：外围 → 大盘 → 板块 → 标的
       │
竞价 (9:15 / 9:20 / 9:25)
  └─→ 验证早评标的：持仓 / 热票竞价是否验证预判
       │
午评 (11:30)
  └─→ 上午实盘验证：早评的判断对了还是错了？
       │
收评 (15:00)
  └─→ 全程总结 + 明日策略 ← 给第二天早评打底
```

**一句话总结：** 早评起框架，竞价抓信号，午评调方向，收评做总结。四条流水线覆盖了交易日全生命周期。

## Skill 层做了什么

`a-stock-advisor/SKILL.md` 规定了三件事：

### 1. 分析模型

- **龙虎榜过滤：** 主力净买 > 3000 万 AND 涨停 AND 占比 < 60% → 资金驱动型标的
- **竞价判断：** 高开 3–5% + 量比 > 1.5 → 强势延续；高开 > 7% + 量比 < 0.8 → 冲高回落预警
- **情绪面：** 涨停/跌停比 > 3 → 高温区；< 0.5 → 冰点期

### 2. 票池推荐管线

龙虎榜延伸 → 机构+北向双买 → 事件催化 → 连板高标延伸 → 持仓替代，按优先级串联。

### 3. 输出协议

Schema 2.0 消息卡片，Markdown 表格直出，不走 `message.presentation`（那个字段已废弃）。

## 持仓同步

用户在飞书说「买了 XXX」或「清了 XXX」→ OpenClaw 写进 `memory/holdings.md` → 四个脚本启动时自动读这个文件，分析自动对齐最新持仓。

负成本 T 仓（做 T 摊到成本为负的票）在脚本里挂零成本做特殊处理。

## 效果

- 竞价盯盘三轮循环，持仓与热票自动五级高亮，注意力集中在真正需要决策的票上
- 早午收评自动推送飞书卡片，每天少花 20–30 分钟手工复盘
- 情绪因子 + 可转债信号并入报告，不再只看裸 K 线和指数
- 板块轮动规则引擎给出结构化判断，收评直接衔接次日早评

## 坑

- **腾讯行情竞价段偶尔拉空** — 集合竞价阶段个股数据响应不稳定。踩坑记录全在 `lessons.md` 里
- **不要依赖 Agent 的「判断」** — 板块推荐和票池只是参考。最终决策用户自己拍板
- **持仓要勤更新** — 忘了说买卖 OpenClaw 就按旧仓出报告。规矩是清仓加仓 30 秒内必须告诉 Agent

## 后续方向

- **Level-2 行情接入** — 换通达信/同花顺逐笔委托接口，竞价阶段资金流向判断更精确
- **飞书台账双向同步** — 让飞书多维表格作为 source of truth，OpenClaw 定期拉 diff 自动更新持仓，消除人力输入误差
- **回测验证** — 现有过滤规则全是启发式的。用 backtrader/zipline 跑一遍历史数据，低胜率规则直接删
- **多 Agent 分工** — 目前单 Agent 包办所有。拆成数据采集 / 分析 / 报告生成三个 Agent，各自选更适合的模型
- **情绪因子深化** — 财联社快讯已接入基础 NER + 情感打分，后续可叠加更多信源做交叉验证
- **止损自动弹窗** — 持仓脚本埋监控 loop，盘中单票跌幅超预设止损线，主动飞书弹 alert 卡片
